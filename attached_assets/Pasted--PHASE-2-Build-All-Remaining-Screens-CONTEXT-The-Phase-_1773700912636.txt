# PHASE 2 — Build All Remaining Screens

## CONTEXT
The Phase 1 UI is built: auth pages, patient dashboard, admin dashboard, appointment booking flow (steps 1-4), patient sidebar layout, admin sidebar layout, and admin patient list. These are UI shells — Supabase wiring will happen later in Cursor/Claude Code.

Now build the remaining screen UIs. Same approach: focus on beautiful, complete UI with proper component structure. Use mock/placeholder data where needed — all Supabase queries will be wired up later.

**REMINDER: Do NOT use Replit's built-in database or Replit Auth. All data references should use the Supabase client pattern already established in the project. If the Supabase client files exist, use them. If not, create placeholder data fetching functions that we'll wire up later.**

**REMINDER: Follow the Pulse & Function design system — forest green (#2C3E2D) primary, warm cream (#F5EDE3) backgrounds, terracotta (#C4884D) accents, pill-shaped buttons, DM Sans headings, warm card borders (#E8E2D8). Do NOT use blue, cold gray, or default shadcn colour scheme.**

## SCREEN 1: Intake Form (/intake)

A digital intake form that patients complete before their first appointment. Single-page form with sections, not a multi-step wizard.

### Layout:
- Page title: "Pre-Appointment Intake Form"
- Subtitle: "Please complete this form before your first consultation. It helps Dr Sarah prepare for your appointment."
- Progress indicator at top showing completion percentage
- Form sections separated by subtle dividers
- Sticky "Save & Submit" button at bottom

### Form Sections:

**Section 1 — Personal Details**
- Full name (text input, pre-filled from registration if available)
- Date of birth (date picker)
- Address (textarea, 2 lines)
- Email (text input, pre-filled)
- Mobile number (text input, pre-filled)

**Section 2 — Emergency Contact**
- Contact name (text input)
- Contact phone number (text input)
- Relationship (dropdown: Spouse/Partner, Parent, Sibling, Friend, Other)

**Section 3 — GP Information**
- GP name (text input)
- GP practice name (text input)
- GP practice address (text input, optional)

**Section 4 — Medical Information**
- Current medications (textarea with placeholder: "List any medications you are currently taking, including dosage if known")
- Known allergies (textarea with placeholder: "List any known allergies — food, medication, environmental")
- Primary health concerns (textarea with placeholder: "What are your main health concerns or reasons for booking this consultation? Please share as much detail as you're comfortable with — this helps Dr Sarah prepare for your appointment." — make this field larger, 4-5 rows minimum)

**Section 5 — Consent**
Three checkboxes, all required:
- [ ] "I agree to the Terms & Conditions" (link to T&C page)
- [ ] "I consent to the processing of my personal data in accordance with the Privacy Policy" (link to Privacy Policy)
- [ ] "I understand that Dr Sarah Al-Temimi is not acting as a replacement for my regular GP and will not be prescribing medication during nutritional consultations"

### Behaviour:
- Form auto-saves on field blur (show small "Saved" indicator)
- Validation: all required fields checked before submission
- On submit: show success state — "Thank you! Your intake form has been submitted. Dr Sarah will review it before your appointment."
- If already completed: show read-only view of submitted form with an "Edit" button

## SCREEN 2: Food Diary (/food-diary)

An interactive 7-day food diary. This is NOT a simple form — it's a structured daily logging tool.

### Layout:
- Page title: "7-Day Food Diary"
- Subtitle: "Record your meals, drinks, and symptoms for 7 consecutive days. Be honest — don't change your diet for the diary! Dr Sarah will review this before your consultation."
- Day selector: horizontal tab/pill bar showing Day 1 through Day 7, with dates shown (e.g., "Day 1 — Mon 17 Mar"). Active day is highlighted. Completed days show a green checkmark. Incomplete days show a hollow circle.
- Progress bar below the tabs: "3 of 7 days completed"

### Per-Day Form (shown when a day tab is selected):
Each day has these meal slots, displayed as stacked cards:

**Breakfast**
- Time (time picker, e.g., "08:30")
- What did you eat? (text input, generous width)
- Approximate amount (text input — e.g., "2 slices toast, 1 cup coffee")
- Location (dropdown pill selector: At Home, Restaurant/Café, Takeaway, At Desk, Other)

**Morning Snack**
- Same fields as breakfast (time, what, amount, location)
- "No snack" toggle — if toggled, collapses the fields

**Lunch**
- Same fields (time, what, amount, location)

**Afternoon Snack**
- Same fields with "No snack" toggle

**Dinner**
- Same fields (time, what, amount, location)

**Evening Snack**
- Same fields with "No snack" toggle

**Additional Drinks**
- Text area: "List any additional drinks throughout the day (water, tea, coffee, juice, alcohol, etc.) with approximate quantities"

**Symptoms**
- Text area: "Note any symptoms you experienced today — digestive issues, energy levels, mood, headaches, sleep quality, etc. Brief notes are fine."

### Design Details:
- Each meal slot is a card with a subtle left border in forest green
- Collapsed "No snack" state shows a thin gray card with "No snack" text
- Auto-save on every field blur with subtle "Saved" toast
- "Mark Day as Complete" button at the bottom of each day
- When all 7 days are complete, show a summary view and a "Submit to Dr Sarah" button
- Empty state for new diary: "Start your food diary by selecting Day 1 and recording your meals."

## SCREEN 3: Documents (/documents)

Where patients view their lab results and upload their own files.

### Layout:
- Page title: "My Documents"
- Filter pills at top: All | Lab Results | My Uploads | Food Diaries | Intake Forms
- Upload button (top right): "Upload Document" — opens a modal or drawer

### Document List:
Each document is a card/row showing:
- File type icon (different icon for PDF, image, Word doc)
- Type badge (e.g., "Lab Result" in green pill, "My Upload" in warm pill)
- Lab provider badge if applicable (e.g., "TDL" or "GutID" or "TeamGene")
- File name
- Date uploaded
- Uploaded by (Dr Sarah or Self)
- Notes from Dr Sarah (if any, shown as a small quote below the file name)
- Download button (icon)
- View button (for PDFs/images, opens in a modal or new tab)

### Upload Modal:
- Drag-and-drop zone with "Drop files here or click to browse" text
- File type info: "Accepted: PDF, JPG, PNG, DOCX — Max 25MB"
- Optional "Notes" text field (for context about what the file is)
- Upload button

### Empty State:
"No documents yet. Your lab results and reports will appear here once Dr Sarah uploads them. You can also upload your own medical records or previous test results."

## SCREEN 4: Messages (/messages)

Two states depending on membership status.

### State A — Member (has active membership):
Full messaging interface.

**Layout:**
- Left side (hidden on mobile): conversation metadata — "Conversation with Dr Sarah", patient's membership badge, conversation status
- Main area: message thread, scrollable, newest at bottom
- Bottom: message input bar

**Message Thread:**
- Messages from patient: right-aligned, forest green background (#2C3E2D), cream text
- Messages from Dr Sarah: left-aligned, cream background (#FAF6EF), dark text, with small "Dr Sarah" label and avatar
- Each message shows timestamp (relative: "2 hours ago", "Yesterday at 3:15 PM")
- File attachments shown as clickable cards below the message text (file name + size + download icon)
- "Read" indicator (subtle checkmarks) on patient's messages

**Message Input:**
- Text input stretching full width
- Attachment button (paperclip icon) — opens file picker
- Send button (arrow icon, forest green)
- Above the input: "Messages are typically responded to within 24–48 hours during business hours. This is not for medical emergencies."

**Loading state:** Skeleton message bubbles

### State B — Non-member:
- Clean card in the centre of the page
- Icon: locked message icon
- Title: "Direct Messaging with Dr Sarah"
- Body: "Messaging is available to membership patients. As a member, you can message Dr Sarah directly for follow-up questions, supplement guidance, and ongoing support between consultations."
- CTA button: "Explore Membership Plans →" (links to membership/pricing page)
- Below: "Already have questions? Book a follow-up consultation instead →" (links to booking)

## SCREEN 7: Admin Patient Detail (/admin/patients/[id])

The most complex admin screen. Comprehensive view of a single patient.

### Layout:
- Back button: "← All Patients"
- Patient header: Name (large), email, phone, membership badge, "Last seen: [date]"
- Tabbed navigation below the header

### Tabs:

**Overview Tab:**
- Contact details card (name, email, phone, address, DOB)
- Membership status card (tier, status, credits used/total, billing dates)
- Next appointment card
- Recent activity timeline (last 5-10 events: appointment booked, form submitted, message sent, document uploaded)

**Appointments Tab:**
- List of all appointments for this patient (upcoming + past)
- Each shows: date, time, type, format, status, admin notes
- "Add Notes" action on each past appointment (textarea that saves)

**Documents Tab:**
- Same document list as the patient sees, but with an additional "Upload Lab Results" button
- Upload Lab Results form: file upload + lab provider dropdown (TDL, GutID, TeamGene) + notes field + toggle "Notify patient about new results"
- Existing documents with delete option

**Messages Tab:**
- Full conversation thread with this patient (same as patient's message view but from admin perspective)
- Message input with reply + file attachment
- Quick-reply dropdown: select from saved templates to insert text
- Conversation status selector: Active / Needs Follow-Up / Resolved

**Supplements Tab:**
- List of current supplement recommendations for this patient
- "Add Recommendation" button → modal with:
  - Dropdown to select from the supplement catalogue (pre-loaded brands)
  - Or "Custom supplement" option with manual name/brand/URL fields
  - Dosage instructions (text input)
  - Notes for patient (text area)
- Each existing recommendation: Edit / Remove actions
- Toggle "Active" status on each supplement

**Notes Tab:**
- Private clinical notes (NOT visible to patient)
- Chronological list of dated notes
- "Add Note" button with text area and date picker
- Each note shows date, content, and edit/delete actions

**Forms Tab:**
- Intake form: shows completed form data in read-only format, or "Not yet submitted" status
- Food diaries: list of submitted food diaries with view button → opens read-only diary view

4. Messages (both patient + admin views) — most impactful feature

For each screen: build the complete UI with mock data, proper loading skeletons, empty states, and mobile responsiveness. Follow the Pulse & Function design system (forest green, warm cream, terracotta accents, pill buttons, warm borders).
